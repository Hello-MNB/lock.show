## 1. What's in this release (changelog since rel-2026.07.09)

**Features**
- **Public Passport split into two buyer personas** — Booking view (event booker) + Representation view (agency). Separate files, one shared firewall-safe data layer; hero toggle + `?view=rep` deep link. (`270dde8`)
- **Artist-controlled WhatsApp sharing** — the artist decides in personal settings whether their number is shared on the request screen. (`d2ee4c8`)
- **V1 · M1 analytics** — the funnel + Gate events now persist to `analytics_event`. (`b06440e`)

**Fixes**
- **🔒 Firewall breach closed (P0)** — unreviewed AI claims could reach the public Passport because the `artist_approved` publish gate was never enforced on any read path. Added `artist_approved = true` to the row gate on all four surfaces: **migration 031** (anon RLS) + `db.js` (authenticated + snapshot) + `server/index.js` (service-role). Found by this release's flow-gap audit.
- **Booker navigation gap** — bookers landed on `/discover` with only an Account tab; added a Passports tab. Nav contract 34/34. (`2e2196f`)
- **Signup dead-ends** — already-registered email now routes to Login (prefilled) instead of a dead end (`65f7703`); signup name split into First + Last (EN/HE) (`33a68f2`).
- **PWA stale-cache trap** — self-destroying service worker un-traps users stuck on old cached app. (`6205ad2`)

**Design**
- Adopted **Design System v1.2.0** as repo SSOT; accent lime → `#C8F04D`; marketing site aligned (Step 1, site-first). (`67f0320`, `b3866e2`)

**Ops / channels**
- Wired official LOCK channels (social, WhatsApp, 11 role inboxes) into site footer + JSON-LD; filled legal email placeholders. (`9d59493`, `a0d6b80`)
- Documented the deploy-hook routine (both Vercel projects). (`c4430d3`, `a90deb2`)
