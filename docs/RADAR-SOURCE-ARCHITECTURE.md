## 2 · ADOPTED — the four source display types (binds Radar UI + Codex iconography)
| Display type | Treatment | Examples |
|---|---|---|
| Branded platform | official logo | Spotify · Instagram · Eventer · Resident Advisor |
| Document/artifact | generic file icon | Ticket Export · Settlement · Rider · Stage Plot |
| Counterparty/entity | role/entity icon | Venue · Event Producer · Manager Office · Label |
| Declared information | LOCK self-reported icon | Fee Band · Community Band · Format |
Law: **a platform logo is never evidence quality** — the method label carries quality; the logo
only says where it came from. Evidence ceilings per source stay per the discovery spec.

## 3 · INTERNAL weight model (never rendered)
The per-entity macro allocation (Artist/Manager/Producer per category, summing 100%) is adopted
as the INTERNAL prioritization layer that: orders next-actions per viewer-entity value · tunes
genre emphasis (with DS v1.6.11 genre table) · informs CFRO's value analysis. It never renders
as numbers anywhere in any UI, artist- or buyer-facing.
