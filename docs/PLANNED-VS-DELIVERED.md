## Summary

Roughly half of what's on today's board is real and correctly wired (firewall, honest confirms,
real Perlman data, GA4, notifications, the top-right workspace switcher's placement, legal-page
scaffolding) — the visual/copy/token layer of today's design corrections landed solidly. But the
owner's actual complaint — multi-Act identity at the center-star, genre tags, the IA nav
correction, and the workspace switcher *doing* something — is the deeper architectural half of
the DESIGN-SPEC rounds, and that half is still sitting in the prototype HTML or half-wired in the
React app while its surface skin got the restraint/master-class pass anyway.

**Single biggest systemic cause:** visual/copy agents (restraint, honesty, tokens) are shipping
daily and outrunning the IA/architecture agent (routing, entity-model, Act CRUD) — so screens now
*look* like the corrected canon while still *behaving* like the pre-correction sitemap underneath.

**Recommended port order:** (1) nav taxonomy fix — collapse Readiness into a Radar mode, add
Requests, stop mislabeling the Claims route "Passport" (cheap, fixes 3 rows above at once); (2)
make the workspace switcher recompute role/home on switch (unblocks Representation/Production
work once 027 is approved); (3) port `applyAct()` + genre tags from the prototype onto
`CenterStar` (schema already exists, migration 020, zero new migration needed) — this is the
owner's most-repeated ask and the cheapest of the three to close; (4) wire `.bloom-confirm` onto
the actual confirm action (one line).
