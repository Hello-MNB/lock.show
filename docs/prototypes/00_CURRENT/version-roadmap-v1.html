1 · The entity model (how LOCK is structured) — vocabulary ratified 12 Jul (DS v1.5.8)

Person → Workspace → Role. Person = the login, never the public artist identity. Workspace = operating context (data, billing, seats); role = permission inside it, not identity. Customer workspaces: Artist (private growth/Radar context; holds Acts — each Act has its own evidence + Passport) · Manager office / אמרגן · משרד אמרגנות (ARTIST-SIDE representation; solo OR team; roster; membership ≠ ArtistAccess) · Production workspace / משרד הפקה (solo OR team; events+lineups).
