## 1 · The checklist (apply to EVERY message, caption, and page of the pack)

### A. Lexicon (hard — ENTITY-GLOSSARY v1.5.8, binding)
- [ ] **פספורט, never דרכון.** The artist artifact is הפספורט / פספורט מקצועי. דרכון is a retired
      term — one occurrence anywhere in the pack = FAIL.
- [ ] **מזמין הופעות, never אמרגן, for the buyer/demand side.** אמרגן = artist-side agency
      (supply side) and is ONLY correct when the message is addressed TO manager offices
      (משרדי אמרגנות) about their own roster. Any use of אמרגן to mean the person who books/hires
      the artist = domain inversion = FAIL.
- [ ] Demand-side segmentation respected (GLOSSARY §2b): professional buyer / private client
      (לקוח פרטי) / corporate (לקוח עסקי) / planner / production — private-register copy is warm
      and non-industry ("מזמינים אמן להופעה?"), never venue jargon at private clients.
- [ ] Other entity words, if used, match GLOSSARY §2: אקט (Act) · מאשר-מקור (Source Confirmer —
      never "מפיק מאשר" in copy) · משרד הפקה for production. No bare "מפיק" for a buyer.

### B. Firewall (hard — CLAUDE.md, absolute)
- [ ] **No score / percentile / rank / rating / "bookability %" / prediction / gauge / grade** —
      in any language, including soft synonyms: "דירוג", "ציון", "מדד", "top artists", "the best",
      "we evaluate/rate artists", star-ratings, leaderboards.
- [ ] **No comparison language** between artists: no "compare artists", "השוואה בין אמנים",
      "see who's stronger", "stand out ABOVE others". Allowed register: proof / confidence /
      verified evidence / method labels ("evidence, not hype").
- [ ] Draw/audience claims, if mentioned at all, only as bands + binaries with method labels —
      never numbers, counts, or percentages promised to buyers.
- [ ] No guarantee language: LOCK is proof/risk-reduction, NOT a booking guarantee, NOT an EPK,
      NOT a CRM. No "get booked more — guaranteed".

### C. Free-pilot posture (hard — ORG-CONSTITUTION + charter)
- [ ] **Zero payment mention.** No price, no "free for now", no "later it will cost", no discount,
      no premium tier, no trial framing. The pilot is simply free — pricing is UNPUBLISHED and the
      commercial Gate is never conflated with launch payment.
- [ ] No monetisation promises to recipients (no "early users get founder pricing" etc.).

### D. Tracking (hard — SYNC §26 convention)
- [ ] Every join link carries a batch token in the **`?src=seed-NN`** format (NN = two-digit
      batch number, e.g. `lock.show/artists?src=seed-01`), on the correct audience page
      (`/artists` · `/managers` · `/production` · `/bookers` · `/producers`).
- [ ] One token per send-batch, unique, never reused across batches; token appears in the pack's
      send-list next to its audience so signups resolve to role + source_page + batch.
- [ ] Links verified WORKING on the live site (Claude verifies before Maria sends — §26 law).

### E. Multi-style / multi-Act framing (hard — CLAUDE.md MULTI-ACT + GLOSSARY §2c)
- [ ] **Multi-style friendly:** an artist with several styles/Acts (psytrance Act + techno Act) is
      framed as NORMAL and supported — never asked to "pick your one genre", never dimmed/shamed
      for breadth. Genres = attributes, not entities; one artist, one account, many Acts.
- [ ] No copy implying evidence or reputation transfers between Acts ("your techno proof boosts
      your psytrance profile" = FAIL — evidence is per-Act, non-transferable).

### F. Tone and truth (soft — flag, may pass with note)
- [ ] Artist-facing: pride-safe, "supported, not inspected"; buyer-facing: risk-to-their-name
      relief, standardized method-labeled evidence.
- [ ] Nothing claimed that isn't live (audience pages + waitlist ARE live; app features described
      only as they exist on production — check DEPLOY-GAPS before claiming).
- [ ] HE is native-register (HE Copy Master polish), paste-ready for WhatsApp/DM, sender = Maria.
